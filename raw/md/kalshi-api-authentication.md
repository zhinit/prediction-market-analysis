# Kalshi API Authentication

Source: https://docs.kalshi.com/getting_started/api_keys, https://docs.kalshi.com/getting_started/quick_start_authenticated_requests
Fetched: 2026-06-24

## API Key Generation

Users access Account Settings, navigate to the Profile Settings page at https://kalshi.com/account/profile, and click "Create New API Key" to generate credentials in RSA_PRIVATE_KEY format.

Upon creation, users receive:
- **Private Key**: Secret key in RSA_PRIVATE_KEY format
- **Key ID**: Unique identifier for the private key

Critical note: "the private key will not be stored by our service, and you will not be able to retrieve it again once this page is closed."

## Required Authentication Headers

Every authenticated API request requires three headers:

1. `KALSHI-ACCESS-KEY` — contains the Key ID
2. `KALSHI-ACCESS-TIMESTAMP` — request timestamp in milliseconds
3. `KALSHI-ACCESS-SIGNATURE` — request hash signed with the private key

## Signing Process

The signature is generated by signing concatenated values: timestamp + HTTP method + path (without query parameters).

**Critical warning**: "use the path **without query parameters**. For example, if your request is to `/trade-api/v2/portfolio/orders?limit=5`, sign only `/trade-api/v2/portfolio/orders`"

**Algorithm**: RSA-PSS with SHA256

## Python Implementation

```python
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import padding
import base64
import time
import requests

# Load PEM-formatted private key
with open("key.pem", "rb") as f:
    private_key = serialization.load_pem_private_key(f.read(), password=None)

# Generate signature
timestamp = str(int(time.time() * 1000))
method = "GET"
path = "/trade-api/v2/portfolio/balance"
message = timestamp + method + path

signature = private_key.sign(
    message.encode("utf-8"),
    padding.PSS(
        mgf=padding.MGF1(hashes.SHA256()),
        salt_length=padding.PSS.MAX_LENGTH
    ),
    hashes.SHA256()
)

headers = {
    "KALSHI-ACCESS-KEY": "<your-key-id>",
    "KALSHI-ACCESS-TIMESTAMP": timestamp,
    "KALSHI-ACCESS-SIGNATURE": base64.b64encode(signature).decode("utf-8")
}

response = requests.get(
    "https://external-api.kalshi.com/trade-api/v2/portfolio/balance",
    headers=headers
)
```

## JavaScript Implementation

```javascript
const crypto = require('crypto');
const fs = require('fs');
const axios = require('axios');

const privateKey = fs.readFileSync('key.pem', 'utf8');
const timestamp = Date.now().toString();
const method = 'GET';
const path = '/trade-api/v2/portfolio/balance';
const message = timestamp + method + path;

const signature = crypto.sign('sha256', Buffer.from(message), {
    key: privateKey,
    padding: crypto.constants.RSA_PKCS1_PSS_PADDING
});

const headers = {
    'KALSHI-ACCESS-KEY': '<your-key-id>',
    'KALSHI-ACCESS-TIMESTAMP': timestamp,
    'KALSHI-ACCESS-SIGNATURE': signature.toString('base64')
};
```

## WebSocket Authentication

WebSocket connections use the same signing mechanism during the handshake. Sign the string: timestamp + "GET" + "/trade-api/ws/v2", then pass the three headers during connection.

## Troubleshooting

Common issues:
- Authorization errors from incorrect key or signing
- Timestamp formatting (must be milliseconds)
- Path construction errors (must exclude query parameters)
- Query parameter accidentally included in signature
